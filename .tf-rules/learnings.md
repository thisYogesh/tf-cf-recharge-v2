## Consolidated
- CRITICAL: Recharge Affinity portal DOM is app-injected React at runtime — it is NOT in any theme file and cannot be introspected from the repo. Style/reorder hooks MUST be verified against the rendered outerHTML (ask the user to copy it from DevTools). Do NOT assume `.recharge-section-{name}` wrapper classes exist — in this store's Affinity version they DON'T; that selector family matches nothing.
- The homepage "next order" card is ONE widget: `[data-widget-type="recharge_next_order"]` / `[data-testid="recharge_next_order"]`. It contains BOTH the next-order actions AND the "In your order" line items — not two separate sections. Stable hooks are `data-testid` attrs (hashed classes like `_17o99wpmo` are volatile — never target them): `next-order-header` (title + action buttons), `next-order-actions` (both CTA buttons), `next-order-date`, `next-order-line-items` ("In your order"), `line-item-image-N`; alert = `.recharge-alert` / `[role="alert"]`.
- Base `.recharge-*` classes DO exist and are styleable: `.recharge-card`, `.recharge-alert`, `.recharge-button-primary`/`-secondary` (+ `.recharge-button`), `.recharge-heading`/`-h2`/`-h4`, `.recharge-text`/`-p3`/`-p4`/`-p5`, `.recharge-image`, `.recharge-badge`, `.recharge-icon`.
- Native next_order render order = header(title+buttons) -> alert -> line-items. Figma wants alert -> title -> [Send Now|Reschedule full-width row] -> "In your order". Fix via JS in recharge-portal.liquid: re-parent [alert, header, actions, lineItems] (by data-testid) directly into the container in order, add class `.rc-next-order-flat` (flex column) to it, hide leftover emptied wrappers. Must be idempotent (bail when order[] are already the trailing children AND extras hidden) + MutationObserver-driven + attempt cap (~60) to avoid fighting React; reset cap on Recharge::location::change / Recharge::action::orderChanged. Buttons keep working after the move (React-Aria/root delegation).
- Tracker widget injects into `*.sidebar` slot; upsell placeholder into `overview.footer` slot.
